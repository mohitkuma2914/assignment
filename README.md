# Backend Engineering Assignment

## 📌 Overview

This repository contains my solutions to a backend engineering assignment focused on core backend concepts, system design, and architectural decision-making.

The assignment evaluates:

* Data modeling (SQL vs NoSQL)
* API design (REST vs GraphQL)
* Authentication & Authorization (JWT, RBAC)
* Microservices vs Monolithic architecture
* Caching strategies

## 🧠 Key Highlights

### 1. Data Storage Design

* Designed schema for both PostgreSQL and MongoDB
* Compared scalability, consistency, and performance
* Justified database selection based on use case

### 2. API Optimization

* Explained limitations of REST (over-fetching, under-fetching)
* Proposed GraphQL-based solution
* Designed migration strategy without breaking existing APIs

### 3. Authentication & Authorization

* Detailed JWT lifecycle
* Secure token handling (refresh + revocation)
* Implemented Role-Based Access Control (RBAC)

### 4. System Architecture

* Compared Microservices vs Monolith
* Real-world use case with trade-offs
* Highlighted scalability and deployment advantages

### 5. Caching Strategies

* Client-side, Server-side (Redis), and CDN caching
* Performance optimization for blog systems
* Cache invalidation techniques
